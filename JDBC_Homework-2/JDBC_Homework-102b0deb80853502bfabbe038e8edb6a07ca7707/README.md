# Автоматизована робота з H2 БД: Міграції Flyway та Клієнтський CRUD Сервіс

Цей проєкт демонструє сучасний підхід до інтеграції Java-додатків із реляційними базами даних за допомогою класичного інструментарію **JDBC (Java Database Connectivity)** у зв'язці з автоматичним керуванням схемами. 

Програма автоматизує розгортання структури БД за допомогою міграцій, реалізує ізольований шар CRUD-операцій із дворівневою перевіркою даних, а також забезпечує безпечне отримання аналітичної звітності за допомогою DTO-мапінгу.

## ⚙️ Технічні характеристики (Стек)
* **Платформа:** Java 17 або новіша
* **Система збірки:** Gradle (керування залежностями драйвера СУБД та Flyway)
* **Простір збереження даних:** Вбудована СУБД H2 (Embedded-режим, локальна база зберігається у файлі `./testdb.mv.db`)
* **Контроль версій БД:** Flyway Community Edition (версія 9.22.3)
* **Рівень доступу до даних:** Чистий JDBC API (використання параметризованих конструкцій, вичитування згенерованих ключів)

---

## 🛠️ Ключові архітектурні рішення та автоматизація

1. **Еволюція схеми через міграції (Flyway):** Ручний запуск SQL-файлів замінено автоматичним контролем версій. Під час старту додатка потокобезпечний синглтон `Database` запускає Flyway. Той аналізує каталог `src/main/resources/db/migration/` і самостійно накочує:
   * `V1__init_db.sql` — архітектуру таблиць та зв'язків між ними;
   * `V2__populate_db.sql` — стартовий набір демонстраційних записів.
2. **Шар CRUD операцій (ClientService):** Створено повноцінний сервіс для ізольованої роботи з сутністю «Клієнт» (`client`):
   * `create` — реєструє нового клієнта та повертає згенерований базою унікальний ID;
   * `getById` — шукає та повертає назву/ім'я клієнта за його ідентифікатором;
   * `setName` — змінює текстове поле імені для конкретного ID;
   * `deleteById` — безповоротно видаляє запис із таблиці;
   * `listAll` — зчитує всі наявні рядки та трансформує їх у колекцію `List<Client>`.
3. **Захисний бар'єр (Двоєтапна валідація):** Коректність даних контролюється паралельно: на рівні бізнес-логіки Java (обмеження довжини імені від 2 до 1000 символів із генерацією `IllegalArgumentException`) та на рівні обмежень самої СУБД (правило `CHECK (LENGTH(name) >= 2)`).

## 📂 Огляд структури директорій

* `src/main/resources/db/migration/` — версіоновані скрипти для автоматичного розгортання та наповнення бази.
* `src/main/java/org/example/` — логіка додатка:
  * `database/Database.java` — централізований потокобезпечний клас керування підключенням, який додатково ініціює Flyway міграції до моменту відкриття першого з'єднання.
  * `dto/` — набір моделей (Data Transfer Objects) для мапінгу складних аналітичних відповідей бази у зручний для Java вигляд (`Client`, `MaxProjectCountClient`, `LongestProject` тощо).
  * `service/ClientService.java` — сервісний шар для безпечного адміністрування клієнтів із вбудованими перевірками.
  * `service/DatabaseQueryService.java` — аналітичний сервіс, що містить параметризовані методи для побудови складних звітів.
  * `util/SqlFileReader.java` — утиліта для імпорту текстових SQL-запитів у Java-код.

## 🚀 Порядок запуску та демонстрація роботи

Проєкт має лінійний життєвий цикл, який повністю запускається через вхідну точку в класі `Main.java`.

1. Переконайтеся, що Gradle успішно імпортував залежності (зокрема драйвер H2 та ядро Flyway).
2. Запустіть метод `main` у файлі `src/main/java/org/example/Main.java`.

### Що відбуватиметься у консолі під час виконання:

1. **Робота Flyway:** Якщо додаток запускається вперше і файл бази відсутній, Flyway створює внутрішню таблицю метаданих і послідовно виконує скрипти `V1` та `V2`.
2. **Перевірка CRUD:** Додаток продемонструє послідовний цикл роботи сервісу — виведе початковий список, створить тестовий запис, змінить його, а потім проведе чистку (видалення).
3. **Тестування валідації:** Програма навмисно спробує зберегти клієнта з некоректним ім'ям (із одного символу). Код успішно перехопить помилку `IllegalArgumentException` і виведе інформативне попередження, не зупиняючи додаток аварійно.
4. **Виведення аналітики:** Сервіс вибірок виконає складні SELECT-запити через захищені інтерфейси та відобразить структуровані Java-об'єкти у консолі.

## 🔒 Надійність, Безпека та Ресурсний менеджмент

* **Профілактика витоків пам'яті:** Усі операції з JDBC-ресурсами (`PreparedStatement`, `ResultSet`) реалізовані через конструкцію **`try-with-resources`**, що гарантує їхнє закриття навіть у разі виникнення помилок у процесі виконання.
* **Стабільність підключення:** Безпосередній життєвий цикл об'єкта `Connection` інкапсульовано всередині Singleton-класу `Database`, що унеможливлює помилки на кшталт передчасного розірвання з'єднання іншими модулями додатка.
* **Абсолютний захист від ін'єкцій:** Застосування **`PreparedStatement`** для абсолютно всіх динамічних запитів усуває будь-які ризики SQL-ін'єкцій, а також підвищує ефективність взаємодії з H2 завдяки механізму попереднього кешування планів запитів.
