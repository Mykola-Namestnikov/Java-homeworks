# SpaceTravel: Сервіс бронювання квитків для міжпланетних подорожей

Цей репозиторій містить кодову базу бекенд-платформи транспортної компанії **SpaceTravel**, що спеціалізується на організації рейсів між планетами Сонячної системи. 

В основі проєкту лежить надійна реляційна модель даних, розгорнута за допомогою автоматичних міграцій та керована з Java-коду за допомогою сучасної ORM-системи.

## 💻 Склад технологічного стека
* **Платформа:** Java (версія 17 або новіша)
* **Збірка та управління:** Інструмент автоматизації Gradle
* **Об'єктно-реляційне відображення:** Hibernate ORM (версія 6)
* **Контроль схем даних:** Flyway (автоматичне керування міграціями)
* **База даних:** H2 (використовує файловий режим для збереження стану між запусками)
* **Оптимізація коду:** Бібліотека Lombok (генерація рутинного коду: геттери, сеттери, конструктори)

---

## 🛠️ Схема та правила валідації даних

Архітектура бази даних складається з трьох ключових сутностей, пов'язаних між собою на рівні констрейнтів:

* **Клієнт (`Client`):** * Ідентифікується за допомогою сурогатного первинного ключа з автоінкрементом (`ID`).
  * Поле імені має обов'язкове обмеження по довжині: щонайменше 3 та не більше 200 символів.
* **Планета (`Planet`):** * Використовує унікальний текстовий ідентифікатор як первинний ключ, який може складатися виключно з цифр та великих літер латинського алфавіту (наприклад, `EARTH1` або `MARS`).
  * Текстова назва планети повинна містити від 1 до 500 символів.
* **Квиток (`Ticket`):** * Фіксує точний час оформлення бронювання (`created_at`) у міжнародному часовому стандарті UTC.
  * Реалізує зв'язки типу **Many-to-One (`@ManyToOne`)**, поєднуючи запис про квиток із конкретним пасажиром (`Client`), а також двома точками маршруту — планетою вильоту та планетою призначення (`Planet`).

## 🚀 Порядок локального розгортання

Завдяки повній автоматизації через Gradle та вбудовану базу даних, проєкт не потребує попереднього встановлення та налаштування сторонніх систем керування БД.

1. Завантажте або склонуйте сирцевий код проєкту на локальний комп'ютер.
2. Перевірте, що у вашій системі налаштована змінна оточення для роботи з Java 17 (або вище).
3. Відкрийте термінал у кореневому каталозі проєкту та виконайте стандартну команду для збірки й запуску додатка:

```bash
./gradlew run
```

### Що відбудеться під час першого старту:

* Автоматичний менеджмент БД: Flyway самостійно створить локальний файл бази даних H2 та послідовно накотить усі наявні SQL-скрипти міграцій для формування таблиць і зв'язків.
* Ініціалізація ORM: Hibernate автоматично підключиться до налаштованої схеми, перевірить відповідність Java-сутностей структурам у базі даних та запустить демонстраційну бізнес-логіку додатка.
