# Консольний Java-додаток для роботи з H2 БД через JDBC

Цей репозиторій містить практичну реалізацію взаємодії Java-додатка з реляційною системою керування базами даних за допомогою технології **JDBC (Java Database Connectivity)**. 

Програма демонструє повний цикл роботи з даними: автоматичне розгортання структури таблиць «з нуля», первинне заселення бази тестовими записами та вибірку складної аналітики за допомогою безпечних параметризованих запитів.

## 💻 Використані технології

* **Мова розробки:** Java (версії 17 або новішої)
* **Інструмент збірки:** Gradle (усі необхідні бібліотеки, включаючи драйвер H2, прописані в дескрипторі залежностей)
* **База даних:** H2 (працює у вбудованому варіанті, локальний файл збереження даних створюється автоматично за шляхом `./testdb.mv.db`)
* **Робота з БД:** Низькорівневий JDBC API (`Connection`, `PreparedStatement`, `ResultSet`)

## 📂 Огляд компонентів проекту

### 🗄️ SQL Скрипти (папка `sql/`)
* `init_db.sql` — опис архітектури таблиць із захистом від повторного створення.
* `populate_db.sql` — скрипт для очищення наявних даних та додавання свіжих демонстраційних записів.
* Інші `.sql` файли — аналітичні `SELECT` конструкції для формування звітності.

### ☕ Вихідний код Java (папка `src/main/java/org/example/`)

* `Main.java` — головний керуючий клас програми.
* `database/Database.java` — потокобезпечний синглтон, що відповідає за створення та перевикористання єдиного пулу з'єднання з базою.
* `dto/` — набір полегшених класів-моделей (Data Transfer Objects) для конвертації рядків з таблиць у Java-формат.
* `service/` — бізнес-логіка програми, розділена на три сервіси: створення структури (`DatabaseInitService`), наповнення (`DatabasePopulateService`) та виконання аналітичних операцій (`DatabaseQueryService`).
* `util/SqlFileReader.java` — допоміжний інструмент для конвертації SQL-коду з текстових файлів у змінні типу `String`.

## 🚀 Порядок локального запуску

Проект повністю автономний і виконує всі етапи роботи послідовно при запуску головного методу.

1. Переконайтеся, що Gradle завантажив проект та підтягнув залежність `com.h2database:h2`.
2. Запустіть метод `main` у файлі `src/main/java/org/example/Main.java`.

### Що саме виконає програма після старту:

1. **Етап ініціалізації:** `DatabaseInitService` створює у файлі БД необхідні таблиці (`worker`, `client`, `project`, `project_worker`), якщо вони не були створені раніше.
2. **Етап генерації даних:** `DatabasePopulateService` видаляє застарілі записи (для уникнення конфліктів і дублювання ID) та записує новий набір тестових даних.
3. **Етап аналітики:** `DatabaseQueryService` виконує п'ять складних аналітичних звітів і виводить сформовані об'єкти Java в системну консоль.

## 🛠️ Надійність та Безпека

* **Контроль ресурсів:** Щоб уникнути забивання пам'яті та витоку з'єднань, уся робота з об'єктами драйвера (`Statement`, `ResultSet` тощо) загорнута в блоки **`try-with-resources`**, які гарантують їхнє вчасне закриття.
* **Захист від ін'єкцій:** Будь-які запити до бази виконуються виключно через об'єкт **`PreparedStatement`**. Це блокує можливість зловмисного впровадження стороннього SQL-коду та пришвидшує обробку запитів самою СУБД завдяки попередній компіляції.
