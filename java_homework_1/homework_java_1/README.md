# Java Веб-додаток (Java Web Application Homework)

Цей проект є навчальним Java веб-додатком, розробленим у рамках виконання домашнього завдання (`homework_java_1`). Проект використовує стандартну архітектуру Maven для збірки веб-додатків та налаштований для розгортання на сервері сервлетів Apache Tomcat.

## 🛠 Технологічний стек

* **Java**: Мова програмування додатка.
* **Maven**: Інструмент для автоматизації збірки, управління залежностями та життєвим циклом проекту.
* **Apache Tomcat**: Сервер сервлетів та веб-сервер для запуску Java Web компонентів.
* **SmartTomcat**: Плагін для IntelliJ IDEA, що використовується для швидкого локального запуску додатка.

## 📁 Структура проекту

Проект має класичну структуру Maven веб-додатка:

```text
homework_java_1/
│
├── .smarttomcat/              # Локальні налаштування сервера Tomcat для плагіна
│   └── homework_java_1/conf/  # Конфігураційні файли Tomcat (server.xml, web.xml тощо)
├── src/
│   ├── main/
│   │   ├── java/              # Вихідний код Java (Сервлети, Фільтри, Контролери)
│   │   ├── resources/         # Ресурси додатка (конфігурації, логування)
│   │   └── WEB-INF/           # Веб-дескриптори та приватні ресурси додатка
│   │       └── web.xml        # Головний конфігураційний файл веб-додатка
│   └── test/
│       └── java/              # Юніт-тести для перевірки працездатності коду
├── .gitignore                 # Файл для ігнорування системних файлів Git
├── pom.xml                    # Головний конфігураційний файл Maven (залежності та плагіни)
└── README.md                  # Документація проекту (цей файл)
```

## 🚀 Інструкція із запуску

Для успішного запуску проекту вам знадобляться встановлені JDK, Apache Maven, а також IDE (рекомендується IntelliJ IDEA).

### 1. Крок 1: Клонування або розпакування проекту

Переконайтеся, що всі файли проекту розпаковані у робочу директорію.

### 2. Імпорт проекту в IDE

1. Відкрийте IntelliJ IDEA.
2. Оберіть Open (Відкрити) та вкажіть шлях до кореневої папки проекту (де знаходиться файл `pom.xml`).
3. Зачекайте, поки Maven завантажить усі необхідні залежності (процес Maven Sync).

### Крок 3: Налаштування та запуск локального сервера (SmartTomcat)

Проект вже містить готові конфігураційні файли для плагіна SmartTomcat у папці `.smarttomcat` (зокрема налаштовані `server.xml` та `web.xml` для сервера).

1. Переконайтеся, що у вас встановлено плагін SmartTomcat в IntelliJ IDEA (можна знайти через *Settings->Plugins*).
2. У верхній панелі IDE натисніть на випадаючий список конфігурацій запуску та оберіть існуючу конфігурацію SmartTomcat (або створіть нову, вказавши шлях до вашого локального Apache Tomcat та обравши модуль `java_homework_1`).
3. Натисніть кнопку **Run** (зелений трикутник).
4. Після успішного запуску сервера додаток буде доступний у вашому браузері за локальною адресою (за замовчуванням: http://localhost:8080/ або відповідно до налаштувань вашого `server.xml`).

### Крок 4: Збірка проекту через Maven (Опціонально)

Якщо вам потрібно скомпілювати та зібрати проект вручну у формат `.war` файлу, виконайте команду в терміналі додатка:

```bash
mvn clean package
```

Зібраний артефакт з'явиться в папці `target/`.

## Додаткові деталі щодо налаштування:

Папка `.smarttomcat` містить специфічні конфігурації, такі як `catalina.properties`, `context.xml` та `tomcat-users.xml`[cite: 316, 322, 333]. Вони відповідають за ізольоване середовище запуску цього проекту, не порушуючи глобальні налаштування Tomcat на вашому комп'ютері.
